Character Set of JAVA

The valid / recognised set of character used in a particular language is called the character set of that language. Character set means that the character and symbols that a C/C++/JAVA Program can understand and accept. These are grouped to from the the commands, expression, words, statements and the token of Java language.

The character set of java can cantain maximum 2^16(65536) character, because java follows UNICODE.

There are following character used in the character set of 'JAVA' :-

  • Capital letters (A to Z)
  • Small leters (a to z)
  • Digit (0 to 9)
  • Special symbol :- All the keybord keys except alphabet, digits and white-spaces are the special symbols.

    • There are following special Symbol/special Character.

  • White space:-

    • One or more space is considered as white space.The compiler ignores white space unless they are a part of a string constant. White space are used to separate words but can't be embedded in the keywords and identifiers.

    There are following white space character :-

    • Blank space - One character space
    • Horizonted tab ('\t') - Five or Eight character space
    • Vertical tab ('\v') - Five or Eight character space
    • Carriage return ('\r')
    • Line feed/New line('\n')
    • Form feed ('\f') - For big space

    It means any types of space is called white space.

  2. Identifiers

    3. Identifier is the name of variable label, array, function, class, interface or any uses defined data item.

    There are certain rules regarding identifier :-

    • Identifier must be meaningful and unique.
    • It is the combination of alphabets / letters(capital as well as small), digits(0-9), underscore(_), and dollor sign($).
    • First characterof an identifier must be an alphabet or underscore(_) or dollar($) sign. It means digits can't be used as first character of an identifier.
    • No special symbol can be used except underscore (_) abd Dollar($).
    • white spaces are not allowed.
    • Pre-defined keywords, function or we can say pre-defined terms can't be used as a name / identifier.
    • With JDK-8, the use of an underscore by itself as an identifier is not recommended.
  3. Constants

    4. Constants are the fixed values that do not change during execution of program. Constants are specified at the right of assignment operator.

    There are basically three types of constant.

    1. Numeric constant :-

      2. A constant that contain numbers ar works on numbers is called numeric constant.

      There are two types of numeric constent

      1. Integer

        2. A constant which has no fractional part is called integer constant. Any number value is an integer literal.

        1. Decimal constant

          2. It contains digits from 0 to 9, with an optional leading +ve or -ve sign.

          Example:- +34, 124, -65 etc.

        2. Octal constent

          3. It contains digits from 0 to 7 and is preceeded/ leaded with Zero(0).

          Example:- 033, 051, 0332, 0327, etc

        3. Hexa decimal constent

          4. It is Combination of digits from 0 to 9 and letters from A to F / a to f and preceeded with 0X(ZeroX).

          Example :- 0X34, 0X34A, etc.

        4. Binary constent

          5. It contain bits 0,1 and leaded / preceeded with 0b / 0B (Zero b/B). Binary constant is added in JDK-7.

          Example :- 0b1010, 0B01010, etc.
