Features/Buzzword of Java

Sun Microsystem officially describes java by given Buzzwords and that are as Follous :-

  1. Compiler & Interpreted
  2. Platform independent & Portable
  3. Object oriented
  4. Roburt & Security
  5. Distributed
  6. Simple & Small
  7. Multi-threaded
  8. High performance
  9. Dynamic
  1. Compiler & Interpreted:-

    Java is both a compiled and an Interpreted language. Java translates source code into bytecode instructions. As bytecode are not machine instructions. Java Interpreter genertes machine code that can directly be executed by the perticular machine that is running the Java Program.

  2. Platform independent & Portable:-

    The most java programs can be earily moved from one computer to another, anywhare and anytime change and upgrade in OS, processors and system resources will not force any changes in java program.

  3. Object oriented:-

    Almost everything in java is an object. All program code and data reside within objects and classes.

  4. Roburt & Security:-

    Java provides many safeguard to ensure reliable code. It has strict compile time and run time checking for datatypes. Security is used for programming on Internet. Threat of virues and abuse of resources are everywhere. The absence of pointers in java ensures that program can't gain access to memory location without proper authorisation.

  5. Distributed :-

    It is designed as a distributed language for creating application on network. It has ability to share both data and program.

  6. Simple & Small:-

    Java is a simple and small language. Many features of C and C++ that are either redundant or sources of unreliable code are not part of java.

    For Example :- Java doesn't use pointers, preprocessor, header files, goto statements and many others.

  7. Multi-threaded :-

    Multi-threaded means handling multiple task simultaneously. It means we need not wait for the application to finish one task before beginning another.

    For Example :- Listening sound clip while browsing a page at a time.

  8. High performance :-

    Use of intermediate bytecode and multi-threading enhances the overall execution speed of java program.

  9. Dynamic :-

    Java is capable of dynamically linking in new class libraries, methods and objects. It can also the determine the type of class through a query, making it possible to either dynamically link or abort the program, depending on the respons.

Java Virtual Machine(JVM)

Java language is a plateform independent usually referred to as "write once run anywhere (WORD)". This is accomplished by the JVM that runs on the local machine, interprets the java bytecode and converts it into platform spccific machine code. The JVM is invoked dufferently depending on the types of java program. JVM performs the following function.

  • When a class file is executed, JVM loads all requried classes automatically from the local disk or from across the network. This is the function of "class loader" utility of JVM.
  • After loading the required classes JVM verifies to make sure that the classes do not violate any of the basic rules of java language. This is the function of "Bytecode Verifies".
  • Then JVM keeps track of all memory uses. It takes care of memory allocation and also performs the release of memory after the object is no longer needed. This process which manages dereferenced objects is caled Garbage Collection.

Note :-

Within JVM, Just-in-Time(JIT) compilers are used to improve performance. JIT compiler translates bytecode only the first time. If repeated execution of the code is required, it is automatically mapped to the correspoding native machine code. This is especially effective in repetative code, such as loops or recursive functions.
