MULTI-THREADING

The process of creating as well as execut more than one threads simultaneasly/parallely/concurrently is called multi-thread. Multi-threading allows multiple task to excute concurrently within a single program. By using multi-threading,we can create programs showing animation, playing music,display document and download environment simultaneously.

In multi-threading concept the main important element to achieve multi-threading is termed/called athread. Athread is a light weighted process in java, it is specified as a class.

The advantage of multi-threaded program is that, it utilises system resources(Like CPU,Memory,input/output devices etc.) better because other thread can grab the CPU time when one thread execution is blocked. A typical program of java is executed swquentially and is said to be a single thread program. If single thread program execution is blocked while wating for input/output operation, then no other process of program execution is processed.

In java, we can create multiple threads that may or may not share memory. The switching between thread are automatically cassied out by JVM(Java Virtual Machine). When a standalone application is run, a thread is automatically created to execute main() function. This thread is called main thread. The program terminates when the main thread finishes execution and all other thread called child thread are spawned with/form the main thread.

Unlike C and C++, java provides facility for multi-threading. It means,java is a Object oriented programming language that provides facility to crate tread and process them.

In java, there are two ways to create a thread :-
  1. using Thread class
  2. using Runable interface
  1. using Thread class :- In java, the first way to create a thread by dereving/inheriting a class form Thread class and over-riding method run() of Thread class.Thread class is specifier in java.lang package. As we know in java, a user defined thread is a user-defined class.

  2. using Runnable interface :- It is the secound way to create a thread in java. In this way a class implemens utilises Runable interface, and used the method run() of Runable interface.

    In this method/way a thread is invoked/called using the object of Thread class, and the object of user defined thread class is passed as argument in Thread class constructor. It means, start() function is invoked/call with object of Thread class, and the object of user defined thread is passed as argument of Thread class constructor.

Life Cycle of Thread :-The stages/states form the creation of thread to tramination of thread is called the life cycle of thread.

There are following states/stages are considered in life cycle of thread.

  1. new born
  2. runnable
  3. running
  4. blocked
  5. dead/terminate
    1. new :- A process/thread is considered in new state when process creates/borns.
    2. runnable :- A process/thread is considered it runnable state when it is not selected by thread schedular.
    3. running :- A thread is considered in running state when it is selected by thread schedular. In this state, thread is executing.
    4. blocked :- A thread is considered in blocked state when a thread performs an input/output operation.
    5. terminate :- A thread is considered in terminate/dead state when a thread completes its whole work.

Methods of Thread class :-
  1. start() :- It is used to start execution of thread. JVM invokes/calls run method on execution of start() method.
    2. Syntex :-public void start();
  2. run() :- It is used to perform action for thread.
    3. Syntex :- public void run();
  3. getName() :-It returns the name of thread.
    4. Syntex :-public final String getName();
  4. setName() :- It changes the name of executing thread.
    5. Syntex :- public final void setName(String);
  5. getPriority() :- It returns the priority of the thread.
    6. Syntex :-public final int getPriority();
    • MIN_PRIORITY= 1
    • NORM_PRIORITY= 5
    • MAX_PRIORITY= 10
  6. setPriority() :- It is used to set the new priority for thread.
    7. Syntex :-public final void setPriority(int Priority);
  7. getId() :-It returns the id of the thread.
    8. Syntex :-public int getId();
  8. currentThread() :- It returns the reference of current executing thread.
    9. Syntex :-public static Thread currentThread();
  9. sleep() :-It causes the currently running thread to sleep for specified period of time.
    10. Syntex :-public static void sleep(long milisecound);

    public static void sleep(long milisecound,int nanosecound);

  10. join() :-It waits for a thread to die.
    11. Syntex :-public void join ();
