Document/Comment section

This section is used to mack a program more meaningful understandable and readable. This section is not processed/executed by 'c' compiler, it only shows a meaningful message.In software development ,it is good practic to place a documentation /comment section. Documentation/comment section is benificial in creation of new software as well as maintainance of existing software. In 'c' comments are specified between (/*) slashstric and stricslase(*/). 

Syntax:- /*message*/
Example:-
  1. /*program to display a message*/
  2. /*prog to print the sum and product of two nos*/
Note:-
  1. In c, comment can be used for single as well as multiline.
  2. In c, comments can't be nested. It means, a comment cannot be placed within another comment.
Example:-

[/* prog to print the /* sum of two nos */ as well as product*/]->Error.

Header file include section

Header file:-A file having extension .h is called header file. A header file contains the prototype of function, while function are stored in C-library in the form of object file. A prototype is one line informtion regarding function.

There are several built in header file in C. They are as follows:-

  • stdio.h (Standard input output)
  • conio.h (console input output)
  • mayh.h
  • ctype.h (character type)
  • string.h
  • stdio.h (standard library)
  • dos.h
  • process.h
  • graphics.h
