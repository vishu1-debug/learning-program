Applet

Applet are small special kind of java programs that run within a web document/web-page.They are accessed an internet server and transported over the network.
An applet is a small program which is embedded in an HTML page that can be automatically downloaded form web server and run in any java enabled lorows or hot java lorowser of java.Applet are automatically installed and run as a part of HTML document.

    The current use of applet includes following :-
  1. Animated graphics
  2. Video games
  3. On-line test
  4. Databases, Reports,etc
Difference between Java applet and Java Application :-
Java Applet Java Application
  1. java Applet are used for creating dynamic and inteactive web application.
  2. Applets are embedded within web page/HTML page and run under the control of browser.
  3. Applet application has no main() method.
  4. Java Applet is only compiled. it is not executed by interpreter, because it has no main() method. It is only executed by browser.
  5. Java Applet does not use the library of other language such as C and C++.
  1. Java applecation are used to write comma cialapplication.
  2. Java application runs stand-alone with support of java virtual machine(JVM).
  3. Java application support main() method.
  4. Since, java application has main method, so it is both compiled as well as interpreted.
  5. Java application has facility to use the library of other languages.
Types of Applet :-
    There are basically two types of applet :-
  1. Local Applet
  2. Remote Applet
  1. Local Applet -->The applet which resides on local computer and also executer on same computer is called local applet.
  2. Remote Applet -->The applet which resides on remote computer such as web-server and executed on that computer is calleed remote applet.

Applet Lifecycle :-

There are following stages in lifecycle of applet :-

  1. Initialieation state
  2. Running state
  3. Blocked state
  4. Display state
  5. Dead state
  1. Initialisation state :-In this state, the applet is first loaded into the memory. It means initialisation state comes only once in applet lifecycle.

    In this state, the public void init() method is automatically triggered/ invoked call.

    2. Syntex :- public void init()
            {
                _ _ _ _ _ _ _ _
                _ _ _ _ _ _ _ _
            }
  2. Running state :-After initialisation, the applet enters into running state. This state shows that an applet is executing it work.

    In this state, the public void start method is automatically invoked.

    3. Syntex :-public void start()
            {
                _ _ _ _ _ _ _ _ 
                _ _ _ _ _ _ _ _ 
            }
  3. Blocked state :-In this state, applet stops its own work temporarily.

    In this state, the public void stop() method is automatically invoked.

    4. Syntex :- public void stop()
            {
                _ _ _ _ _ _ _
                _ _ _ _ _ _ _
            }
  4. Display state :-It is a past of running state. This state shows that the applet is displaying somthing on the web page.

    In this state, the public void paint() method is automatically invoked.

    5. Syntex :-public void paint(Graphics g)
            {
                _ _ _ _ _ _ _ _
                _ _ _ _ _ _ _ _ 
            }
  5. Dead state :-It happened only once in a applet lifecycle. It has come when either the applet has completed its work, or we exit form the browser or closing the web page.

    In this state, the public void destroy method is automatically invoked.

    6. Syntex :-Public void destroy()
            {
                _ _ _ _ _ _ _  _ _
                _ _ _ _ _ _ _ _  _ 
            }
